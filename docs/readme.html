1       Introduction

This is a general update to the Microchip MPLAB® XC Network Server. This version updates RLM to 14.1(BL3) .

1.1     Previous Versions

The previous MPLAB XC Network Server was version 2.00 which was based on RLM 11.2(BL2) .

1.2     System Requirements

The MPLAB XC Network Server is available for a variety of 64-bit operating systems, including Microsoft® Windows® 10 and above;  Ubuntu® 9.10 and above; or Mac OS® X 10.15 and above. 

The server may also run on various other operating systems including Windows Server, Linux distributions, such as Oracle® Enterprise Linux®  and Red Hat® Enterprise Linux as well as older versions of supported operating systems. However, XC License server is not tested on these operating systems. 

The MPLAB XC Network License Server can be run on Virtual Machines of the supported OS using a virtual machine license for network licenses  (SW006021-VM).

32-bit versions of MPLAB XC Network Server are discontinued starting from version 3.00.

1.3     Installation

Run the .exe (Windows), .run (Linux) or .app (OS X) compiler installer application. For example, XCNetLicServer-3.00-windows.exe and follow the directions on the screen. Using the default installation directory is recommended. If you are using Linux, you must install the compiler using a terminal and from a root account. Install using a Mac OS X account with administrator privileges.

Full information on installation and operation of the MPLAB XC License Server is covered in the newest version of the XC License Server Manual (DS5002334).

2       What’s New

2.1     Version 3.00

   • Updated to the latest version of RLM - MPLAB XC Network Server v3.00 is based on the Reprise License Manager version 14.1(BL3). This upgrade provides various enhancements and bug fixes made by Reprise to RLM license server. 

   • Updated XC Network Server manual - This release includes an updated manual 

   • RLM as windows service can longer be added from web interface - Refer the updated user manual to know how RLM can be added as a Windows service from the command prompt.


2.2     Version 2.00

  • Separate Installer - Previous versions of the MPLAB XC Network Server were installed by selecting the Server Install in one of the MPLAB XC Compiler installers. Starting with version 2.00, the Network Server has its own installer.
  • Updated to the newest version of RLM - MPLAB XC Network Server is based on the Reprise License Manager version 11.2(BL2).
  • Status messages include additional information - The status option for the included xclm command now displays additional information on when the HPA (High Priority Access) for the license expires.
  • New XC Network Server manual - This release includes a new manual that includes information on Installation and Operation of the MPLAB XC Network Server. Also included is a “How To” section that answers commonly asked questions.
  • Warnings are issued when HPA and demo licenses are about to expire - A warning is displayed when the license is within 14 days of the HPA or the demo license expiring.


 3       Migration Issues


3.1     Before migrating from version 2.00 to 3.00

   • Before taking down the v2.00 server, make sure that all the roamed licenses are returned to the server. RLM does not support roaming out licenses from one version and roaming it in on another version. 


4       Fixed Issues

4.1     Version 3.00

XCLM-140: Network server installer fails on some Windows machines - On some Windows machines, the installer aborts with a message saying C:\ProgramData\Microchip is not present

4.2     Version 2.00

XCLM-77: Unable to Roam a license back into the server - If the Windows version of the rlm command is shut down with a license roamed out, then the license server may forget that the license is roamed out and may issue an error if an attempt to return the license occurs before it times out.

XCLM-78: Can’t read license data (-102) error issued when running any xlcm command - The Network Server installer that was included with each compiler was not configuring the xclm command. The new, separate installer used for v2.00 of the Network Server now configures the xclm command

5       Known Issues

5.1     Version 2.00

XCLM-56: Host ID reported by 'xclm -hostinfo' changes on a same computer - If a removable network device (like an Android phone or a WiFi adapter) is recognized as the first MAC address found, it will display as the hostID when xclm -hostinfo runs. If the device is removed, and that hostID was used to activate a license, then the license will not be valid. xclm will always check each MAC address to see if it matches the hostID in the license file, so any fixed MAC address (with the '-'s removed) can be used as the hostID when the license is activated.
